    Sistema de Recomendação de Jogos

    Ceub ~ APS 3° Semestre

    Alunos:

    • Luiz Paulo Raeder
    • Eduardo Rodrigues
    • Douglas Amorim

    Visão Geral e Contexto do Sistema

    O sistema é uma plataforma que ajuda os usuários, por meio de uma api de IA, a descobrir jogos com base nos seus gostos, histórico de navegação, histórico de recomendação e preferências individuais.

    O sistema contará com cinco módulos principais:

    • Interface do Usuário: Módulo para navegação, onde o usuário poderá interagir com a plataforma e explorar as recomendações.
    • Módulo de Avaliações: Processará dados das avaliações dos usuários para armazenar no banco de dados e fornecê-los aos outros usuários.
    • Módulo de Recomendações: Utilizando uma API de inteligência artificial, processará dados do histórico e das preferências do usuário para gerar recomendações personalizadas.
    • API IA: Irá se basear em dados fornecidos pelos outros módulos como histórico, avaliações, recomendações e amizades para que uma recomendação de jogo seja gerada por meio de uma IA.
    • Banco de Dados: Armazenará informações dos jogos e dos perfis dos usuários, facilitando o processamento de recomendações e o histórico.

    Contexto de Aplicação

    O sistema será desenvolvido para atender a um público diverso, desde jogadores casuais os mais experientes que buscam recomendações confiáveis para seu tempo de lazer. A ferramenta também poderá ser útil para empresas desenvolvedoras de jogos, fornecendo insights sobre tendências e preferências do público.

    Em resumo, o sistema de recomendações de jogos representa uma solução moderna que conecta jogadores e contribui para um mercado de entretenimento mais acessível e adaptado às necessidades dos usuários.

    1. Planejamento e Definição de Requisitos

    Objetivo

    Mapear as funcionalidades principais do sistema e definir os requisitos técnicos e funcionais de cada módulo, considerando os recursos disponíveis e o alinhamento com as necessidades do projeto.

    1.1 Requisitos Funcionais

    Os requisitos funcionais definem o comportamento esperado do sistema, detalhando as ações que ele deve realizar para atender às necessidades dos usuários:

    1. Cadastro e Login de Usuários

    2. Sistema de Pesquisa de Jogos

      • Possibilitar a busca de jogos por título, categoria ou desenvolvedor.

    3. Sistema de Recomendação

      • Gerar recomendações personalizadas com base no histórico de navegação e preferências dos usuários utilizando IA.

    4. Interação com Jogos

      • Permitir que os usuários avaliem, favoritem e adicionem jogos à lista de desejos.

    5. Sistema de Amizades

      • Oferecer a funcionalidade de adicionar amigos e visualizar as recomendações deles.

    6. Histórico de Recomendações

      • Armazenar e exibir as recomendações anteriores para consulta futura.

    7. Feedback do Usuário

      • Coletar avaliações e sugestões para melhorar as recomendações.

    1.2 Requisitos Não Funcionais

    Os requisitos não funcionais definem as restrições e características de qualidade que o sistema deve atender:

    1. Desempenho

      • O sistema deve ser capaz de gerar recomendações em no máximo 2 segundos para consultas padrão.

    2. Usabilidade

      • A interface deve ser intuitiva e compatível com dispositivos móveis.

    3. Segurança

      • Garantir a criptografia de dados sensíveis, como senhas e informações pessoais.
      • Implementar controle de acesso para garantir que cada usuário veja apenas seus próprios dados e recomendações.

    4. Escalabilidade

      • O sistema deve ser projetado para suportar o crescimento da base de usuários e o aumento no número de jogos armazenados.

    5. Portabilidade

      • O sistema deve ser compatível com navegadores comuns (Chrome, Firefox, Safari) e ser responsivo em dispositivos móveis.

    6. Manutenibilidade

      • O código deve ser modular e documentado para facilitar futuras atualizações e correções.

    7. Confiabilidade

      • O sistema deve apresentar um bom uptime mensal, com monitoramento para identificar falhas rapidamente.

    8. Integração

      • Deve ser possível integrar o sistema com APIs IA de terceiros para que seja realizado o cálculo de recomendação.

    1.3 Cenário de Caso de Uso (Tabela)

    Caso de Uso Descrição
    Fazer Login Usuário faz login na plataforma com verificação em duas etapas.
    Pesquisar Jogos Permite ao usuário buscar jogos por título, categoria ou desenvolvedor.
    Adicionar à Lista de Desejos Usuário adiciona jogos a uma lista pessoal para acompanhar títulos de interesse.
    Avaliar/Favoritar Jogos Usuário pode avaliar ou favoritar jogos para personalizar recomendações futuras.
    Recomendações Personalizadas Geração de recomendações baseadas no histórico e nas interações do usuário.
    Adicionar Amigo Usuário pode adicionar amigos à sua lista, permitindo interações sociais no sistema.

    5. Diagramas

    5.2 Diagrama de Sequência

    O diagrama de sequência ilustra a interação entre os objetos do sistema ao longo do tempo para realizar funções específicas.

    Exemplo de Caso: Obter Recomendações de Jogos

    1. O usuário solicita recomendações por meio da interface.
    2. A interface envia a solicitação para o módulo de autenticação, verificando a identidade do usuário.
    3. O módulo de autenticação valida o token JWT e direciona a solicitação ao módulo de recomendações.
    4. O módulo de recomendações consulta o banco de dados para obter o histórico do usuário.
    5. Após consultar a API de IA, o sistema retorna uma lista personalizada de jogos.
    6. A interface exibe as recomendações ao usuário.

    5.3 Diagrama de Classes

    O diagrama de classes representa as classes do sistema, suas propriedades, métodos e relacionamentos.

    Estrutura Inicial:

    • Classe Usuario
      • Atributos: id, nome, email, senha, historico
      • Métodos: autenticar(), adicionarAmigo(), avaliarJogo()
    • Classe Jogo
      • Atributos: id, titulo, categoria, desenvolvedor
      • Métodos: obterDetalhes(), adicionarFavorito()
    • Classe Recomendacao
      • Atributos: id, id_usuario, id_jogo, data_recomendacao
      • Métodos: gerarRecomendacoes()
    • Classe BancoDeDados
      • Atributos: conexao
      • Métodos: consultar(), salvar()

    5.5 Diagrama de Pacotes

    O diagrama de pacotes organiza os módulos do sistema, mostrando as dependências entre eles.

    Estrutura de Pacotes:

    • Pacote Principal: Sistema
    • Subpacotes:
      • Interface: Contém a camada de apresentação para o usuário.
      • Autenticacao: Gerencia o login, tokens e permissões.
      • Recomendacao: Integra a IA e processa dados do usuário.
      • BancoDeDados: Implementa operações CRUD para armazenar e recuperar dados.

    Dependências: - A Interface depende da Autenticacao e da Recomendacao. - A Recomendacao depende do BancoDeDados.
